12 — Adicionando um Novo Tema
▸ Processo passo a passo
Passo 1 — Definir a Paleta (7 Decisões)
VariávelO que controlaDica
--bg / --bg2 / --bg3Fundo principal, secundário, terciárioEscuro → texto claro. Claro → texto escuro. Gradação sutil entre os 3.
--stone / --stone2Superfície de cards e inputsDeve ser visivelmente diferente de --bg, mas não chamar atenção.
--divBordas e separadoresMais sutil que --stone. Fronteira invisível mas presente.
--teal (accent primário)Elementos interativos, anel do timer, estados ativosContraste mínimo 3:1 sobre --stone2. Deve "saltar" do fundo.
--gold (accent secundário)Avisos, highlights, botão Resume/RegistrarComplementar ao --teal. Não use a mesma família de cor.
--cream / --sub / --muted / --dimHierarquia de texto (4 níveis)4 passos distintos de contraste. Cream ≥ 7:1, Sub ≥ 4.5:1 sobre --bg.
--green / --red / --errSucesso, erroManter intuição cromática (verde = bom, vermelho = alerta).
Verificar contraste: Use o DevTools → Inspector → computed value para checar a relação de contraste (ícone de olho). WCAG AA exige 4.5:1 para texto normal, 3:1 para texto grande e elementos UI.
Passo 2 — Adicionar ao themes.css

Arquivo: themes.css (raiz do repositório). Adicionar bloco ao final:

html.theme-meu-tema { --bg: #0a0808; /* fundo mais escuro */ --bg2: #110f0d; --bg3: #180f0c; --stone: #1e1510; /* superfície de card */ --stone2: #251a14; /* superfície de input/card aninhado */ --div: #3a2a20; /* bordas, separadores */ --gold: #d8a840; /* accent secundário — warnings, highlights */ --gold2: #8a6020; /* variante de borda dourada */ --teal: #4ac8c0; /* accent primário — interações, anel */ --cyan: #70e0d8; /* variante mais clara do teal */ --green: #18cc30; --red: #c83030; --err: #e05050; --cream: #f0e8d0; /* texto primário */ --sub: #b8a890; /* texto secundário */ --muted: #786858; /* texto terciário */ --dim: #4a3a2a; /* texto mais fraco / placeholder */ --font: 'Cinzel', serif; /* titular, labels, UI */ --font-body: 'Crimson Pro', serif; /* corpo de texto */ }

Nome do tema: kebab-case — ex: sheikah-slate, hylia-light, blossom-rose.

Fonte nova no Google Fonts? Adicionar @import url(...) no topo do themes.css.

Passo 3 — Registrar no designer.html

Adicionar um pill na barra de temas no topo desta página:

<button class="d-th-pill" data-theme="meu-tema" onclick="setTheme('meu-tema')">Meu Tema</button>

Trocar para o novo tema e percorrer todas as seções. Corrigir qualquer variável que pareça errada.

Passo 4 — Registrar no App

Encontrar onde a lista de temas é gerenciada no app (Settings, Nav ou similar) e adicionar a mesma string de chave. O script de FOUC em cada página HTML aplica automaticamente — não é necessário alterar nenhum arquivo HTML.

/* Script FOUC — já existe em todos os .html, não precisa mudar: */ !function(){ var t = localStorage.getItem('cone_theme') || 'totk-dark'; document.documentElement.classList.add('theme-' + t); }();
Passo 5 — Checklist de Validação

Percorra esta página no novo tema e marque cada item:

  • Todos os swatches de cor mostram as cores corretas (sem transparent / variável faltando)
  • Hierarquia de texto legível nos 4 níveis: cream → sub → muted → dim
  • --teal (accent primário) visível sobre fundo --stone2 dos cards
  • --gold (accent secundário) lê-se bem ao lado de texto --cream
  • Botão Start (fundo --teal) — texto preto legível sobre o fundo do botão
  • Botão Resume (fundo --gold) — texto preto legível
  • Borda de foco dos inputs (--teal) visível sobre --stone2
  • Item ativo do nav distingue-se claramente do inativo
  • Pill ativo (teal border + tint) distingue-se do pill inativo
  • Anel do timer (stroke --teal) visível sobre fundo --bg
  • Cards de categoria (border-left colorida) lê-se bem em ambas as variantes dark/light
  • Estados: spinner, empty, error — todos visíveis e com contraste adequado
